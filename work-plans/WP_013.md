# WP_013: Dashboard Charts ‚Äî Accounts, Transactions, Goals, Budget

## Context
The dashboard currently shows only 2 charts (spending pie + budget vs actual bar) plus a text-based goals list. This WP adds 4 new charts ‚Äî one per data domain ‚Äî giving users a richer visual overview directly on the Dashboard page.

---

## New Charts

| # | Title | Type | Data source |
|---|-------|------|-------------|
| 1 | Income vs Expense Trend | Area (2 series) | New backend endpoint `/api/dashboard/trend` |
| 2 | Account Balances | Horizontal Bar | `AccountService.getAll()` |
| 3 | Goals Progress | RadialBar | Existing `summary.goals` |
| 4 | Budget Utilization | Horizontal Bar (% used) | Existing `summary.budgetVsActual` |

---

## Phase 1 ‚Äî Backend

### 1.1 Add `Trend` endpoint to `DashboardController.cs`

**File:** `backend/dotnet/budget-api/Controllers/DashboardController.cs`

New action:
```
GET /api/dashboard/trend?months=6
```

Logic:
- Read `cutoffDay` from `AppSettings` (default 1)
- Start from current period (same logic as `Summary`)
- Loop backwards `months` times, calling existing `GetPeriod(cutoffDay, m, y)` for each
- For each period: query `Transactions` by date range, sum `Income` and `Expense`
- Return array ordered **oldest ‚Üí newest**:
  ```json
  [{ "month": 1, "year": 2026, "label": "Jan 2026", "income": 0, "expense": 0 }]
  ```

### 3.3 `ngOnInit` additions
- Inject `AccountService` alongside existing services
- After `this.load()`: call `loadTrend()` and `loadAccounts()` once (not period-sensitive)

### 3.4 New methods

**`loadTrend()`** ‚Äî calls `svc.getTrend(6)`, builds `trendSeries` + `trendXaxis`

**`loadAccounts()`** ‚Äî calls `accountSvc.getAll()`, builds `accountSeries` + `accountXaxis`

**`buildRadial(data)`** ‚Äî filters goals where `targetAmount > 0`, computes `pct = min((current/target)*100, 100)`, sets `radialSeries` + `radialLabels`; called from existing `load()`

**`buildUtilization(data)`** ‚Äî computes `pct = target > 0 ? (actual/target)*100 : 0` per budget entry, sets `utilSeries` + `utilXaxis`; called from existing `load()`
