# Budget App MVP Plan

## Context
The project is a full-stack budget management app (ASP.NET Core 10 backend + Angular 21 frontend). Both sides are currently empty scaffolds — no domain models, no EF Core, no routes, no UI. This plan builds the complete MVP: a working CRUD app covering all 5 planned features with a connected Angular + API backend. No authentication in the MVP.

---

## Scope

5 features, fully functional (list + create + edit + delete):
- **Categories** — Income or Expense, with Name, Color, Icon
- **Transactions** — Date, Amount, Description, Type, Category, Notes
- **Budgets** — Monthly target per category (Month/Year/TargetAmount)
- **Goals** — Savings goals with progress tracking and status
- **Dashboard** — Summary cards (income vs expense this month), spending-by-category pie chart (ApexCharts), budget-vs-actual bar chart, goals progress list

## Phase 1 — Backend

### 1.1 Add NuGet packages (run from `backend/dotnet/budget-api/`)
```
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools
```

### 1.2 Domain models — create `Models/` folder

**Models/Category.cs**
```csharp
public class Category {
    public int Id { get; set; }
    public string Name { get; set; } = "";
    public string Type { get; set; } = ""; // "Income" | "Expense"
    public string Color { get; set; } = "#000000";
    public string Icon { get; set; } = "";
}
```

**Models/Transaction.cs**
```csharp
public class Transaction {
    public int Id { get; set; }
    public DateOnly Date { get; set; }
    public decimal Amount { get; set; }
    public string Description { get; set; } = "";
    public string Type { get; set; } = ""; // "Income" | "Expense"
    public int CategoryId { get; set; }
    public Category? Category { get; set; }
    public string Notes { get; set; } = "";
}
```

**Models/Budget.cs**
```csharp
public class Budget {
    public int Id { get; set; }
    public int CategoryId { get; set; }
    public Category? Category { get; set; }
    public int Month { get; set; }
    public int Year { get; set; }
    public decimal TargetAmount { get; set; }
}
```

**Models/Goal.cs**
```csharp
public class Goal {
    public int Id { get; set; }
    public string Name { get; set; } = "";
    public string Description { get; set; } = "";
    public decimal TargetAmount { get; set; }
    public decimal CurrentAmount { get; set; }
    public DateOnly Deadline { get; set; }
    public string Status { get; set; } = "Active"; // "Active" | "Completed" | "Paused"
}
```

### 1.3 DbContext — create `Data/BudgetDbContext.cs`
```csharp
public class BudgetDbContext(DbContextOptions<BudgetDbContext> options) : DbContext(options) {
    public DbSet<Category> Categories => Set<Category>();
    public DbSet<Transaction> Transactions => Set<Transaction>();
    public DbSet<Budget> Budgets => Set<Budget>();
    public DbSet<Goal> Goals => Set<Goal>();
}
```

### 1.4 Update `appsettings.json`
Add connection string:
```json
"ConnectionStrings": {
  "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=BudgetApp;Trusted_Connection=True;TrustServerCertificate=True;"
}
```

### 1.5 Update `Program.cs`
Add before `builder.Build()`:
- `builder.Services.AddDbContext<BudgetDbContext>(...)` with SQL Server provider
- `builder.Services.AddCors(...)` allowing `http://localhost:4200`
- `app.UseCors(...)` in the pipeline (before `UseAuthorization`)
- Delete `WeatherForecastController.cs` and `WeatherForecast.cs`

### 1.6 Controllers — create `Controllers/` entries

All controllers follow REST pattern:
- `GET /api/categories` — list all
- `GET /api/categories/{id}` — get one
- `POST /api/categories` — create
- `PUT /api/categories/{id}` — update
- `DELETE /api/categories/{id}` — delete

Same pattern for `/api/transactions`, `/api/budgets`, `/api/goals`.

`TransactionsController` gets an extra query param: `GET /api/transactions?categoryId=&type=&month=&year=` for filtering.

`BudgetsController` gets: `GET /api/budgets?month=&year=` for filtering by period.

`DashboardController` — one endpoint `GET /api/dashboard/summary?month=&year=` returning:
```json
{
  "totalIncome": 0,
  "totalExpense": 0,
  "spendingByCategory": [{ "categoryId", "categoryName", "color", "amount" }],
  "budgetVsActual": [{ "categoryId", "categoryName", "target", "actual" }],
  "goals": [{ "id", "name", "targetAmount", "currentAmount", "status" }]
}
```

### 1.7 Migrations
```
dotnet ef migrations add InitialCreate
dotnet ef database update
```

## Phase 3 — Angular API Services

Create `core/services/api/` with one service per resource:
- `category.service.ts` — CRUD methods calling `/api/categories`
- `transaction.service.ts` — CRUD + filter params
- `budget.service.ts` — CRUD + month/year filter
- `goal.service.ts` — CRUD
- `dashboard.service.ts` — `getSummary(month, year)`

All use `HttpClient` via constructor injection. API base URL `http://localhost:5082` defined as a constant.

## Phase 4 — Feature Views

### Pattern for each feature
Each feature folder (`views/<feature>/`) contains:
- `<feature>.ts` — list page component
- `<feature>-form.ts` — create/edit form (modal or inline)
- `<feature>.html` — template using Bootstrap 5 table/card layout

### Categories page
- Table: Name | Type badge | Color swatch | Icon | Actions
- Modal form: Name (text), Type (Income/Expense select), Color (color input), Icon (text)

### Transactions page
- Table: Date | Description | Category | Type badge | Amount | Actions
- Filter bar: Category dropdown, Type toggle, Month/Year pickers
- Modal form: all fields, Category dropdown populated from categories API

### Budgets page
- Month/Year selector at top
- Table: Category | Target Amount | Actual Spent | Remaining | % bar
- Modal form: Category (dropdown), Month, Year, Target Amount

### Goals page
- Cards layout: Name | Progress bar | Current/Target | Deadline | Status badge | Actions
- Modal form: Name, Description, Target Amount, Current Amount, Deadline, Status

### Dashboard page
- Row 1: 4 summary cards — Total Income, Total Expense, Net Balance, Active Goals
- Row 2: Spending by Category pie chart (ApexCharts `ng-apexcharts`) | Budget vs Actual bar chart
- Row 3: Goals progress list
- Month/Year selector to filter the period

Install `apexcharts` and `ng-apexcharts` for charts:
```
npm install apexcharts ng-apexcharts
```
