# WP_037 — Rebrand BudgetApp → Personal Assistant

## Context

The app started as a budget tracker but has grown into a general personal-management suite: Budget/Transactions, Loans, Vehicles, Books, Grocery, Pantry, Goals, and Credit Cards. "BudgetApp" no longer describes what the product is. The user wants a full rebrand to **Personal Assistant**, confirmed via clarifying questions to cover:

1. **Cosmetic branding** — all user-facing text.
2. **Code-level identifiers** — C# namespace, Angular project name/dist path, localStorage/sessionStorage keys, JWT issuer/audience.
3. **Infrastructure** — deploy script updated with new names for *future* use only; **live Azure resources are NOT recreated/migrated** (resource group `budget-app`, ACR `budgetappacr01`, container apps `budget-api`/`budget-frontend`, SQL DB `BudgetApp` stay as-is).
4. **GitHub repo** — rename `ccassob/budget-app` → `ccassob/personal-assistant`.
5. **Local root folder** — rename `C:\projects\budget-app` → `C:\projects\personal-assistant`, done **last**, since the active session's shell cwd lives inside it.

The in-app "Budget" tab/feature (Transactions sub-nav) keeps its own name — only the product identity changes. The database name (`BudgetApp`, everywhere including local Docker) is intentionally left unchanged, matching the "don't touch live infra" decision and avoiding a `.bacpac` restore.

Note: this repo currently has unrelated in-flight changes already in the working tree (modified `CLAUDE.md`, `docker-compose.yml`, `Dockerfile`, `angular.json`, deleted `nginx.conf`; untracked `default.conf.template`, `environment.azure.ts`, `deploy-azure-containers.ps1`, deploy logs, a `.bacpac`). The rebrand will be committed as its own logical set of changes, not mixed into those.

## Execution Order

**Phase 1 — Backend code identifiers** (atomic: all land together or `dotnet build` breaks)
1. `git mv backend/dotnet/budget-api backend/dotnet/personal-assistant-api`
2. `git mv .../budget-api.csproj .../personal-assistant-api.csproj`; update `RootNamespace`
3. `git mv .../Data/BudgetDbContext.cs .../Data/PersonalAssistantDbContext.cs`
4. `git mv backend/dotnet/budget-api.Tests backend/dotnet/personal-assistant-api.Tests`
5. `git mv .../budget-api.Tests.csproj .../personal-assistant-api.Tests.csproj`; update `RootNamespace` and the `<ProjectReference>` relative path
6. `git mv .../BudgetApiFactory.cs .../PersonalAssistantApiFactory.cs`
7. Scripted literal-token replace across `git ls-files "backend/dotnet/*.cs"` (~120 files), in this order to avoid double-replacement:
   - `budget_api.Tests` → `personal_assistant_api.Tests`
   - `budget_api` → `personal_assistant_api`
   - `BudgetDbContext` → `PersonalAssistantDbContext`
   - `BudgetApiFactory` → `PersonalAssistantApiFactory`
8. Update `appsettings.json` `Jwt:Issuer` / `Jwt:Audience`
9. Verify: `dotnet build`, `dotnet test`
10. Spot-check `Migrations/BudgetDbContextModelSnapshot.cs` and any `*.Designer.cs` — their `[DbContext(typeof(...))]` attribute and class body get the token replace too. Leave the **filename** `BudgetDbContextModelSnapshot.cs` as-is; EF Core renames it automatically on the next `dotnet ef migrations add` — forcing it now is unnecessary risk for a harmless, self-correcting mismatch.

**Phase 2 — Frontend code identifiers** (atomic: angular.json + package.json + Dockerfile + folder rename must land together, or the Docker build breaks)
1. `git mv frontend/angular/budget-front frontend/angular/personal-assistant-front`
2. `angular.json` project key rename
3. `package.json` `name` normalize; `README.md` heading
4. `Dockerfile` line 11 `COPY --from=build /app/dist/budget-front/browser` → new dist path
5. Cosmetic text: `index.html` `<title>`, `public/manifest.webmanifest` (name/short_name/description), `constants/index.ts` `APP_NAME`, `views/auth/auth.ts` login heading, `app.ts` title suffix
6. Storage key renames: `auth.service.ts`, `layout.service.ts`, `constants/index.ts`, `views/settings/settings.ts` (4 occurrences)
7. Verify: `npm install`, `ng build --configuration production`, `ng build --configuration docker`, `ng test`

Phases 1 and 2 are independent and can be done in either order, but each must be internally atomic.

**Phase 3 — Infra/deploy/docs**
1. `docker-compose.yml` — build context paths only (leave `Database=BudgetApp` alone)
2. `deploy-azure-containers.ps1` — variable values + docstring (not executed)
3. `publish.ps1` — `$root`, `$frontendSrc`, `$backendSrc`, `.csproj` filename, `Get-Process -Name` list, dist copy path
4. `iis-setup.ps1` — `$outputDir`, `$frontSrc`/`$apiSrc`, `$frontDest`/`$apiDest` (inetpub paths), site/pool names
5. Root `CLAUDE.md` — heading, prose, all path/namespace/JWT/DbContext/`.http` filename mentions; leave the "Budget" sidebar-label mention alone
6. Nested `frontend/.../CLAUDE.md` — dist path + storage key mentions
7. `git mv budget-api.http personal-assistant-api.http` (referenced by root `CLAUDE.md`)

**Phase 4 — GitHub repo rename** (after Phase 1–3 are committed and pushed)
```
gh repo rename personal-assistant --repo ccassob/budget-app
git remote set-url origin https://github.com/ccassob/personal-assistant.git
```

**Phase 5 — Local root folder rename (LAST)**
```
Rename-Item C:\projects\budget-app C:\projects\personal-assistant
```
Update `publish.ps1` `$root` and `iis-setup.ps1` `$outputDir` to the new absolute path as part of this step (they can't correctly reference the new location until after the rename happens). Re-verify `dotnet build` / `ng build` / `docker compose up --build` from the new path afterward.

## Consequences (flagged explicitly, not silently fixed)

- **All logged-in users are force-logged-out** after this deploys: JWT issuer/audience change invalidates existing tokens, and the `TOKEN_KEY` rename orphans the old `budget_token` entry regardless.
- **Theme preference resets** for existing sessions (`__BUDGET_THEME__` → new key, sessionStorage so low impact).
- **PWA/mobile users with a custom API URL override lose it** until they re-enter it in Settings (`__BUDGET_API_URL__` → new key).
- **Docker images must be rebuilt from scratch**: `docker compose up --build`, not `up` — cached layers reference the old dist path.
- **IIS local deployment breaks**: the current live IIS app-pool/physical-path setup points at `C:\projects\budget-app` (per project memory) — Phase 5's rename breaks that binding; it needs manual repointing, not fixed by this plan. Running the *updated* `iis-setup.ps1` afterward creates **new** sites/pools (`personal-assistant-front`/`-api`) without removing the old `budget-front`/`budget-api` ones or their `C:\inetpub` folders — those need manual cleanup via IIS Manager if no longer wanted.
- **Live Azure App Settings diverge from local config**: the deployed container apps' env vars still say `budget-api`/`budget-frontend` (JWT issuer/audience) since live infra isn't touched — intentional, but flagged so it isn't mistaken for a bug later.
