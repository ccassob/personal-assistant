# WP_035 — Dockerize Local Development Stack

## What was done

Replaced the fragile manual LocalDB + `ng serve` local setup with a full Docker Compose stack.

## First-time data import (LocalDB → container)

```powershell
# 1. Install SqlPackage if not present
dotnet tool install -g microsoft.sqlpackage

# 2. Export existing LocalDB data
SqlPackage /Action:Export `
  /SourceConnectionString:"Server=(localdb)\MSSQLLocalDB;Database=BudgetApp;Trusted_Connection=True;TrustServerCertificate=True;" `
  /TargetFile:BudgetApp.bacpac

# 3. Start only SQL Server and wait for it to be healthy
docker compose up -d sql
# Wait ~30s for SQL Server to be ready

# 4. Import the backup (replace SA_PASSWORD with value from your .env)
SqlPackage /Action:Import `
  /SourceFile:BudgetApp.bacpac `
  /TargetConnectionString:"Server=localhost,1433;Database=BudgetApp;User Id=sa;Password=<your-SA_PASSWORD-value>;Encrypt=False;TrustServerCertificate=True;"

# 5. Start the rest
docker compose up -d --build
```

## Day-to-day usage

```powershell
docker compose up -d        # start everything (images already built)
docker compose up -d --build # rebuild images then start
docker compose down         # stop (data volume preserved)
docker compose down -v      # stop AND wipe the database
docker compose logs -f api  # tail API logs
```

## Notes

- The API auto-applies EF migrations on startup; after a `.bacpac` import it is a safe no-op.
- Azure production deployment (App Service + Static Web App + Azure SQL) is untouched.
- Integration tests continue using InMemoryDatabase and are unaffected.
